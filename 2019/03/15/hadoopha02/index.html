软件规划:

统一放在:

/home/hadoop/software/

3.1 添加用户:

[root@bigdata003 ~]# useradd hadoop

3.2 切换用户:

[root@bigdata003 ~]# su - hadoop

3.3 创建目录结构:

[hadoop@bigdata003 ~]$ mkdir software app tmp data log lib shell sourcecode
[hadoop@bigdata003 ~]$

3.4 添加hosts主机名ip:

[root@bigdata003 ~]# vi /etc/hosts

127.0.0.1 localhost localhost.localdomain localhost4 localhost4.localdomain4
::1 localhost localhost.localdomain localhost6 localhost6.localdomain6
172.16.xxx.22 bigdata001
172.16.xxx.21 bigdata002
172.16.xxx23 bigdata003

注意:这里配置内网ip,集群之间通信传输速度很快,否则走公网速度很慢的.三台都要配置!

3..5 配置ssh信任关系

切换用户:

su - hadoop

ll -a一下,如果有.ssh文件则先备份一下

mv .ssh .ssh.bak

生成ssh:

[hadoop@bigdata001 ~]$ ssh-keygen

三台都执行,生成秘钥

然后到第二三台服务器的目录下把id_rsa.pub文件cp到第一台服务器

第二台服务器:

cd .ssh

[hadoop@bigdata002 .ssh]$ scp id_rsa.pub root@bigdata001:/home/hadoop/.ssh/id_rsa.pub2

第三台服务器:

cd .ssh

[hadoop@bigdata003 .ssh]$ scp id_rsa.pub root@bigdata001:/home/hadoop/.ssh/id_rsa.pub3

以第一台为主,把第二,三台的公钥给第一台,使集群能够保持通信.

第一台服务器执行:

[hadoop@bigdata001 .ssh]$ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
[hadoop@bigdata001 .ssh]$ cat ~/.ssh/id_rsa.pub2 >> ~/.ssh/authorized_keys
[hadoop@bigdata001 .ssh]$ cat ~/.ssh/id_rsa.pub3 >> ~/.ssh/authorized_keys

修改权限:

[hadoop@bigdata001 .ssh]$ chmod 0600 ~/.ssh/authorized_keys

再把第一台的authorized_keys文件传给第二三台,sz/rz、FTP等方式都可以，如果用root传的记得修改用户.

初始化:

三台机器分别执行

[hadoop@bigdata001 .ssh]$ ssh bigdata001 date

[hadoop@bigdata001 .ssh]$ ssh bigdata002 date

[hadoop@bigdata001 .ssh]$ ssh bigdata003 date

第一次输入yes是 在known_hosts文件有记录，下次不需要再输入yes.

ll一下看known_hosts文件是否生成.

3.6 JDK部署

切换到root用户

解压:

[root@bigdata001 ~]# cd /home/hadoop/software

tar -zxvf jdk-8u45-linux-x64.gz -C /usr/java

修改用户:

[root@bigdata001 java]# cd /usr/java
[root@bigdata001 java]# chown -R root:root jdk1.8.0_45

配置环境变量:

vi /etc/profile

在最后追加:

export JAVA_HOME=/usr/java/jdk1.8.0_45
export PATH=$JAVA_HOME/bin:$PATH

source生效:

source /etc/profile

tips:曾经改/etc/profile不小心手抖PATH变量写错了,然后有的命令执行不了,source不能生效,可以直接先执行export PATH=/usr/bin:/usr/sbin:/bin:/sbin,再去修改source生效即可.

如果用的阿里云主机防火墙可以不用关闭,但是端口号记得开启安全组才可以访问,自己电脑的虚拟机记得关闭防火墙.

执行命:service iptables stop

验证:service iptables status

3.7 部署Zookeeper

切换用户:

su - hadoop

cd software

tar -zxvf zookeeper-3.4.5-cdh5.16.2.tar.gz -C ../app/

软连接:

cd app

ln -s zookeeper-3.4.5-cdh5.16.2 zookeeper

(软连接的好处可以在本博客搜)

cd zookeeper/conf

修改配置:

cp zoo_sample.cfg zoo.cfg

vi zoo.cfg

修改dataDir=/home/hadoop/tmp/zookeeper,为什么不用Linux的tmp目录,也可以在本博客搜.

追加

server.1=bigdata001:2888:3888
server.2=bigdata002:2888:3888
server.3=bigdata003:2888:3888

创建目录:

mkdir -p /home/hadoop/tmp/zookeeper

touch /home/hadoop/tmp/zookeeper/myid

echo 1 > /home/hadoop/tmp/zookeeper/myid(其他两台追加2/3即可)

启动服务:

[hadoop@bigdata001 bin]$ sh zkServer.sh start

查看状态:

[hadoop@bigdata001 bin]$ sh zkServer.sh status

注意:三台都启动才可以看到zk的状态!

初始化:

先启动日志进程:

[hadoop@bigdata001 sbin]$ cd /home/hadoop/app/hadoop/sbin

sh hadoop-daemon.sh start journalnode

jps查看进程是否存在

在第一台机器上格式化:

[hadoop@bigdata001 sbin]$ cd ../bin

[hadoop@bigdata001 bin]$ sh hadoop namenode -format

把第一台初始化好的元数据传给第二三台,传输的路径为初始化好后打印出来的路径

[hadoop@bigdata001 bin]$ scp -r /home/hadoop/data/dfs hadoop@bigdata002:/home/hadoop/data

[hadoop@bigdata001 bin]$ scp -r /home/hadoop/data/dfs hadoop@bigdata003:/home/hadoop/data

ZKFC初始化:

[hadoop@bigdata001 bin]$ sh hdfs zkfc -formatZK

zk只需在一台机器上执行初始化即可.

启动:

[hadoop@bigdata001 bin]$ cd ../sbin

先关一下进程:

[hadoop@bigdata001 sbin]$ sh stop-all.sh

再启进程:

[hadoop@bigdata001 sbin]$ sh start-all.sh

注意这里启动的顺序,namenode–>datanode–>journalnode–>ZKFC

jps查看进程:

备机的resourcemanager需要手动启动:

[hadoop@bigdata002 sbin]$ sh yarn-daemon.sh start resourcemanager

与预期进程一致,部署完成!
