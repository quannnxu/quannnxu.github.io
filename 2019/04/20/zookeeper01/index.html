Zookeeper

1.解压

2.修改配置

cd conf

cp zoo_sample.cfg zoo.cfg

vi zoo.cfg

修改:dataDir=$ZOOKEEPER_HOME/tmp/zookeeper

环境变量加上zookeeper

3.启动

启动zk进程:

./zkServer.sh start

停止zk进程:

./zkServer.sh stop

进入客户端:

./zkCli.sh

节点在ZK中叫做znode,分为临时节点和永久节点.

临时节点:

1.当前session有效,关闭会被删除

2.临时节点下不能有子节点

3.临时有序

永久节点:

1.不会被自动删除

2.可以有子节点

3.有序

znode特点:

1.每一个znode都有一个身份id、版本号、修改/删除如果版本号不匹配,会报错.

2.ZK的数据一定是比较小的

3.设置权限ACL

4.为znode添加监听器

由此可见,删除也可以监听,但监听只能使用一次.

四字命令:

https://zookeeper.apache.org/doc/r3.4.14/zookeeperAdmin.html#sc_zkCommands

需在zoo.cfg加上:

4lw.commands.whitelist=*

再重启ZK进程就可以使用了.

public class ZKUtils &#123;
    ZooKeeper zk = null;
    String connect ="warehouse001:2181";
    int timeout = 5000;
    String path = "/testpath";

    public static ZooKeeper getZK(String connect, int timeout) throws Exception &#123;
        return new ZooKeeper(connect,timeout,null);
    &#125;

    public static String createNode(String path, String value, ZooKeeper zk) throws Exception &#123;
        return zk.create(path,value.getBytes(), ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
    &#125;

    public static String getData(String path, ZooKeeper zk) throws Exception &#123;
        return new String(zk.getData(path,false,null));
    &#125;

    public static void updataData(String path, String value, ZooKeeper zk) throws Exception &#123;
        zk.setData(path, value.getBytes(), -1);
    &#125;

    public static boolean exists(String path, ZooKeeper zk) throws Exception &#123;
        Stat exists = zk.exists(path, false);
        return exists != null;
    &#125;

    public static List getChildren(String path, ZooKeeper zk) throws Exception &#123;
        return zk.getChildren(path, false);
    &#125;

    public static void delete(String path, ZooKeeper zk) throws Exception &#123;
        zk.delete(path, -1);
    &#125;
