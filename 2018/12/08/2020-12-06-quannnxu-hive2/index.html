Hive基础之Hive参数配置及语法
2018-12-08
Hadoop
Hadoop

1.参数修改

1.1获取hive执行引擎:

set hive.execution.engine;

1.2修改hive执行引擎:

set hive.execution.engine=tez;(看情况修改)

1.3修改hive-site.xml

打印头信息:

hive.cli.print.header
true

显示最近使用的库:

hive.cli.print.current.db
true

显示如下:

hive (default)> select * from test1;
OK
test1.id
100
Time taken: 0.035 seconds, Fetched: 1 row(s)

2.Hive中参数设置的几种方式:

​ hive-site.xml 全局

​ hive –hiveconf

​ 命令行set a=b;

优先级:hive-site.xml < hive –hiveconf < set a=b

3.Hive命令行

​ -e：SQL语句
​ -f：SQL文件
​ –hiveconf：设置参数
​ -i：定义UDF函数
​ exit

3.1 hive -e使用:

[hadoop@warehouse001 bin]$ hive -e “select * from test1”;
OK
test1.id
100
Time taken: 3.248 seconds, Fetched: 1 row(s)

hive -e以后shell封装脚本使用

3.2 hive -f使用:

[hadoop@warehouse001 bin]$ vi 1.sql

select * from test1;

[hadoop@warehouse001 bin]$ hive -f 1.sql
OK
test1.id
100
Time taken: 3.363 seconds, Fetched: 1 row(s)

4.Hive SQL

https://cwiki.apache.org/confluence/display/Hive/LanguageManual+DDL官方语法,本blog语法来自官方文档

Create Database

CREATE (DATABASE|SCHEMA) [IF NOT EXISTS] database_name
  [COMMENT database_comment]    --描述信息
  [LOCATION hdfs_path]    --自定义HDFS路径
  [MANAGEDLOCATION hdfs_path]
  [WITH DBPROPERTIES (property_name=property_value, ...)];

如:

create database hive;

HDFS路径:/user/hive/warehouse/hive.db

元数据库DBS表信息

Drop Database

DROP (DATABASE|SCHEMA) [IF EXISTS] database_name [RESTRICT|CASCADE];

hive (hive)> show tables;
OK
tab_name
test
Time taken: 0.018 seconds, Fetched: 1 row(s)
hive (hive)> drop database hive;
FAILED: Execution Error, return code 1 from org.apache.hadoop.hive.ql.exec.DDLTask. InvalidOperationException(message:Database hive is not empty. One or more tables exist.)

当数据库有表时不能直接删除,可使用CASCADE强制级联删除,但不推荐使用.

Alter Database

ALTER (DATABASE|SCHEMA) database_name SET DBPROPERTIES (property_name=property_value, ...);   -- (Note: SCHEMA added in Hive 0.14.0)

ALTER (DATABASE|SCHEMA) database_name SET OWNER [USER|ROLE] user_or_role;   -- (Note: Hive 0.13.0 and later; SCHEMA added in Hive 0.14.0)

ALTER (DATABASE|SCHEMA) database_name SET LOCATION hdfs_path; -- (Note: Hive 2.2.1, 2.4.0 and later)

ALTER (DATABASE|SCHEMA) database_name SET MANAGEDLOCATION hdfs_path; -- (Note: Hive 4.0.0 and later)

Use Database

USE database_name;
USE DEFAULT;

Create Table

CREATE [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.]table_name    -- (Note: TEMPORARY available in Hive 0.14.0 and later)
  [(col_name data_type [column_constraint_specification] [COMMENT col_comment], ... [constraint_specification])]
  [COMMENT table_comment]
  [PARTITIONED BY (col_name data_type [COMMENT col_comment], ...)]
  [CLUSTERED BY (col_name, col_name, ...) [SORTED BY (col_name [ASC|DESC], ...)] INTO num_buckets BUCKETS]
  [SKEWED BY (col_name, col_name, ...)                  -- (Note: Available in Hive 0.10.0 and later)]
     ON ((col_value, col_value, ...), (col_value, col_value, ...), ...)
     [STORED AS DIRECTORIES]
  [
   [ROW FORMAT row_format] 
   [STORED AS file_format]
     | STORED BY 'storage.handler.class.name' [WITH SERDEPROPERTIES (...)]  -- (Note: Available in Hive 0.6.0 and later)
  ]
  [LOCATION hdfs_path]
  [TBLPROPERTIES (property_name=property_value, ...)]   -- (Note: Available in Hive 0.6.0 and later)
  [AS select_statement];   -- (Note: Available in Hive 0.5.0 and later; not supported for external tables)
Storage Format Description
STORED AS TEXTFILE Stored as plain text files. TEXTFILE is the default file format, unless the configuration parameter hive.default.fileformat has a different setting.Use the DELIMITED clause to read delimited files.Enable escaping for the delimiter characters by using the ‘ESCAPED BY’ clause (such as ESCAPED BY ‘') Escaping is needed if you want to work with data that can contain these delimiter characters. A custom NULL format can also be specified using the ‘NULL DEFINED AS’ clause (default is ‘\N’). (Hive 4.0) All BINARY columns in the table are assumed to be base64 encoded. To read the data as raw bytes:TBLPROPERTIES (“hive.serialization.decode.binary.as.base64”=”false”)
STORED AS SEQUENCEFILE Stored as compressed Sequence File.
STORED AS ORC Stored as ORC file format. Supports ACID Transactions & Cost-based Optimizer (CBO). Stores column-level metadata.
STORED AS PARQUET Stored as Parquet format for the Parquet columnar storage format in Hive 0.13.0 and later; Use ROW FORMAT SERDE … STORED AS INPUTFORMAT … OUTPUTFORMAT syntax … in Hive 0.10, 0.11, or 0.12.
STORED AS AVRO Stored as Avro format in Hive 0.14.0 and later (see Avro SerDe).
STORED AS RCFILE Stored as Record Columnar File format.
STORED AS JSONFILE Stored as Json file format in Hive 4.0.0 and later.
STORED BY Stored by a non-native table format. To create or link to a non-native table, for example a table backed by HBase or Druid or Accumulo. See StorageHandlers for more information on this option.
INPUTFORMAT and OUTPUTFORMAT in the file_format to specify the name of a corresponding InputFormat and OutputFormat class as a string literal. For example, ‘org.apache.hadoop.hive.contrib.fileformat.base64.Base64TextInputFormat’. For LZO compression, the values to use are ‘INPUTFORMAT “com.hadoop.mapred.DeprecatedLzoTextInputFormat” OUTPUTFORMAT “org.apache.hadoop.hive.ql.io.HiveIgnoreKeyTextOutputFormat”‘ (see LZO Compression).

如:

create table table_name(

id int,name string,age int

)

CREATE TABLE：指定要创建的表的名字
col_name data_type：列名以及对应的数据类型，多个列之间使用逗号分隔
PARTITIONED BY：指定分区
CLUSTERED BY： 排序、分桶
ROW FORMAT：指定数据的分隔符等信息
STORED AS：指定表存储的数据格式：textfile rc orc parquet等
LOCATION：指定表在文件系统上的存储路径
AS select_statement: 通过select的sql语句的结果来创建表

内部表和外部表的区别:

​ 内部表:创建语法create table xxxx;

​ 内部表删除表之后HDFS和元数据都被删除了

​ 外部表:创建语法create external table xxx;

​ 外部表删除之后只删除元数据,HDFS文件依旧还存在
